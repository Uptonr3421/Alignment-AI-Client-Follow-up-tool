# Contributing

Thank you for your interest in contributing to the Cleveland LGBTQ Center Client Automation project.

## Project Scope

This project was built specifically for the Cleveland LGBTQ Center. While we welcome adaptations for other nonprofits, the core features should remain focused on client follow-up automation for mission-driven organizations.

## How to Contribute

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature`
3. **Make your changes**
4. **Run tests**: `npm run test`
5. **Run linting**: `npm run lint`
6. **Submit a pull request**

## Code Standards

- TypeScript strict mode
- ESLint and Prettier compliance
- WCAG 2.1 AA accessibility
- Clear comments explaining "why" not "what"

## Documentation

Update relevant docs when adding features:
- User-facing changes → USER_GUIDE.md
- API changes → API_REFERENCE.md
- Setup changes → QUICK_START.md
